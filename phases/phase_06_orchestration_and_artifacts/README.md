# Phase 06 - Orchestration And Artifacts

Purpose: execute the complete weekly workflow, support scheduled triggering, and package deliverables.

## What Phase 6 Does
- Runs phases in sequence (01 -> 06).
- Applies user-selected week range filter (`1..15`) after ingestion.
- Supports partial runs with `--from-step` and `--to-step`.
- Supports skip flags (`--skip-email`, `--skip-artifacts`).
- Writes run metadata (`pipeline_run_meta.json`) with step status and duration.
- Supports weekly scheduled trigger with dedupe guard (one successful send per ISO week).
- Packages latest outputs into `artifacts/latest` and writes `manifest.json`.

## Commands

Manual full pipeline:
- `python phases/phase_06_orchestration_and_artifacts/run_weekly_pipeline.py --app-id com.nextbillion.groww --delivery-mode manual_cli --weeks-from 1 --weeks-to 15`

Manual full pipeline with email dry run:
- `python phases/phase_06_orchestration_and_artifacts/run_weekly_pipeline.py --app-id com.nextbillion.groww --delivery-mode manual_cli --weeks-from 1 --weeks-to 15 --email-dry-run`

Partial run (example: regenerate note -> email -> artifacts):
- `python phases/phase_06_orchestration_and_artifacts/run_weekly_pipeline.py --from-step phase_04_weekly_note --to-step phase_06_artifacts --delivery-mode manual_cli`

Scheduled trigger (intended Monday 10:00 AM):
- `python phases/phase_06_orchestration_and_artifacts/send_weekly_scheduled.py`

Scheduled trigger test bypassing time window:
- `python phases/phase_06_orchestration_and_artifacts/send_weekly_scheduled.py --force --email-dry-run`

## Week Range Rules
- `--weeks-from` and `--weeks-to` are both limited to `1..15`.
- `weeks_from <= weeks_to` is required.
- Week 1 means most recent week.
- Selected range is applied by:
  - `apply_week_range_filter.py`

## Schedule Behavior
- Weekly recipient source: `WEEKLY_SCHEDULED_RECIPIENT`.
- Schedule keys: `WEEKLY_SCHEDULE_DAY`, `WEEKLY_SCHEDULE_TIME`.
- Optional weekly range keys: `WEEKLY_WEEKS_FROM`, `WEEKLY_WEEKS_TO`.
- Dedupe state file: `data/outputs/scheduled_state.json`.
- Skips duplicate weekly send if already completed for current ISO week.
- Use `--no-dedupe` only when duplicate run is intentionally required.

## Run Metadata
- Manual/default run metadata:
  - `data/outputs/pipeline_run_meta.json`
- Scheduled run metadata:
  - `data/outputs/pipeline_run_meta_scheduled.json`

Each metadata file includes:
- start/end timestamps
- selected steps
- week range
- step-wise duration/status
- final run status

GitHub scheduler:
- Workflow: `.github/workflows/weekly_backend_trigger.yml`
- Cron: Monday 10:00 AM IST (`30 4 * * 1` UTC)
- Calls backend trigger URL with bearer token.
